# Dashboard

**Route:** `/dashboard`
**Auth required:** Yes
**Layout group:** `(app)`

The first screen after login. Orients the user toward resuming or validating strategy work â€” not a generic greeting. The primary action is always deterministic and derived from live strategy state.

## Sections

### 1. Strategy workspace hero
```
Strategy workspace
Backtest activity and readiness
Learn about assumptions â†’
```
The hero shows a fixed lab-language heading. A prominent primary CTA button is always visible (unless data is still loading), selected by `selectNextAction(strategies)` via this deterministic rule:

1. If any non-archived strategy's `last_run_at` is newer than its `updated_at` â†’ **"Review last backtest"** (links to that strategy's backtest view).
2. Else if any strategy has `last_run_at == null` â†’ **"Continue editing {name}"** (links to the most-recently-updated draft's strategy page).
3. Else â†’ **"Run a backtest"** (links to the most-recently-updated strategy's backtest view).
4. If zero non-archived strategies exist â†’ **"Create a strategy"** (links to `/strategies`).

### 2. Signal cards row (3-column grid, collapses to 1 col on mobile)

| Card | Icon | Data source | Signal |
|---|---|---|---|
| **Drafts / unvalidated** | `Layers` | `GET /strategies/` | Count of non-archived strategies where `last_run_at == null` |
| **Last run** | `Activity` | `GET /strategies/` | `latest_total_return_pct` + `latest_max_drawdown_pct` of the strategy with the most recent `last_run_at`; links to that strategy's backtest view |
| **Data freshness** | `Clock` | `GET /strategies/` | Relative time since the most recent `last_run_at` or `last_auto_run_at` across the workspace |

Cards show a `<Skeleton>` placeholder while data is loading. The Last run and Data freshness cards show a `â€”` / "No runs yet" empty state when no backtest results exist.

### 3. Recent Backtests *(conditional)*
Reads recent backtest refs from `localStorage` via `getRecentBacktests()`, then fetches each run. Shows a list card with:
- Date range (`date_from` â†’ `date_to`)
- Asset badge and timeframe
- `total_return_pct` with colour-coded up/down arrow

### 4. Your Strategies
Shows up to 5 strategies from `GET /strategies/` with clone quick action. States:

| State | UI |
|---|---|
| **Loading** | 3 skeleton rows |
| **Empty** | Centred empty state with a "Create Strategy" CTA |
| **Populated** | Strategy rows with name, asset badge, last-updated timestamp, and "Clone" button |

## Data fetching

| Source | Endpoint | Notes |
|---|---|---|
| Strategies list | `GET /strategies/` | Full list; first 5 shown in Your Strategies; signals computed client-side |
| Recent backtests | `GET /backtests/:runId` Ă— N | Fetched in parallel; errors silently skipped |

All fetches on mount via `useEffect`. Clone action calls `POST /strategies/:id/duplicate` then refreshes the list.
