# Dashboard

**Route:** `/dashboard`
**Auth required:** Yes
**Layout group:** `(app)`

The first screen after login. It helps the user resume work by surfacing the next strategy to backtest, the latest result to review, or the first strategy creation step.

---

## Sections

### 1. Page heading
```
Next strategy check, <displayName>
Validate saved changes, then review the latest result.
```
`displayName` is derived from the user's email prefix, then cleaned for readability.

The heading includes a secondary link to `/how-backtests-work` labelled "Backtest assumptions".

### 2. Next action panel

Raised card that chooses the most useful next action from the current strategy and recent backtest data.

| Condition | Title | Action |
|---|---|---|
| Strategy load failed | Strategy data did not load | Retry loading |
| No strategies | Create your first strategy | Create strategy |
| Strategy needs testing | Test `<strategyName>` | Run backtest |
| Recent completed backtest | Review the `<asset>` result | Open result |
| Current results available | Continue from `<strategyName>` | Open strategy |

The panel also shows three decision signals:

- Validation queue (`Clear` or `<count> to validate`)
- Latest outcome
- Untested work (`None`, `<count> untested`, `<count> changed`, or both)

### 3. Your Strategies
Shows up to 5 strategies from `GET /strategies/` with a duplicate quick action. States:

| State | UI |
|---|---|
| **Loading** | 3 skeleton rows |
| **Empty** | Centered empty state with a "Create strategy" CTA |
| **Populated** | Strategy rows with name, asset badge, timeframe, last-updated timestamp, and "Duplicate" button |

---

## Data fetching

| Source | Endpoint | Notes |
|---|---|---|
| Strategies list | `GET /strategies/` | Full list; first 5 shown in the table |
| Recent backtests | `GET /backtests/:runId` x N | Used only to choose the latest review action and outcome; errors silently skipped |

All fetches on mount via `useEffect`. The duplicate action calls `POST /strategies/:id/duplicate` then refreshes the list.

---

## Key components used

- `Card`, `CardContent`, `Badge`, `Skeleton`, `Button` (shadcn/ui)
- `useAuth`, `useDisplay` contexts
- `getRecentBacktests` (localStorage helper)
- `formatDateTime`, `formatPercent` (format lib)

| Breakpoint | Layout |
|---|---|

| `sm` (640px) | Next action metrics use 3 columns |
| `md` (768px) | Strategy list switches to table-like rows |
| `lg` (1024px) | Heading and next action controls align horizontally |
