# Strategies List

**Route:** `/strategies`
**Auth required:** Yes
**Layout group:** `(app)`

The primary workspace management screen. Shows all of the user's strategies with search, sort, filter, import/export, and bulk management. Also the entry point for the **Strategy Wizard** (new-user onboarding flow).

---

## Entry modes

| Mode | Trigger | Behaviour |
|---|---|---|
| **Normal** | Direct navigation | Full strategies list |
| **Wizard** | `?wizard=true` query param | `StrategyWizard` modal is rendered on top of the list; marks onboarding complete on close |

---

## Toolbar

```
[New Strategy ‚ñº] [Import] [Templates]   [Search‚Ä¶]  [Asset ‚ñº] [Sort ‚ñº] [Performance ‚ñº] [Last Run ‚ñº]
```

- **New Strategy** ‚Äî opens `NewStrategyModal`
- **Import** ‚Äî triggers JSON file picker; validates, then `POST /strategies/import`
- **Templates** ‚Äî navigates to `/strategies/templates`
- **Search** ‚Äî client-side text filter on strategy `name`
- **Asset filter** ‚Äî filters by `ALLOWED_ASSETS` (BTC/USDT, ETH/USDT, ‚Ä¶)
- **Sort** ‚Äî name, updated_at, total_return, last_run, asset; toggles asc/desc
- **Performance filter** ‚Äî all / positive / negative (based on last backtest return)
- **Last Run filter** ‚Äî all / within 7 days / within 30 days / never run

---

## Strategy table

Columns: **Name**, **Asset**, **Timeframe**, **Last Run**, **Return**, **Actions (‚Ä¶)**

Actions dropdown per row:
- Edit (open canvas)
- Duplicate (`POST /strategies/:id/duplicate`)
- Export (downloads JSON)
- Delete (`DELETE /strategies/:id` with confirmation dialog)

---

## Strategy states

| State | UI |
|---|---|
| Loading | Skeleton rows in the table |
| Empty (no strategies) | Centred empty state with wizard CTA |
| Empty (filtered) | "No strategies match your filters" message |
| Error | Destructive banner with error message |
| Limit reached | Warning badge; "New Strategy" button shows usage tooltip |

---

## Usage limits

Reads `user.usage` from `useAuth`. When `usage.strategies_used >= usage.strategies_limit`, the New Strategy button shows a "Limit reached" tooltip and the modal is blocked.

---

## New Strategy Modal

Fields: **Name** (required), **Asset** (select), **Timeframe** (select).

On submit: `POST /strategies/` ‚Üí navigates to `/strategies/:id`.
