## Security Risks

### HIGH — Redis 7.1.1 — DarkReplica CVE-2026-23631 (4th day unpatched)

**CVE-2026-23631 (DarkReplica):** Post-authentication RCE via replica sync. An attacker who can authenticate to a Redis instance can instruct the server to become a replica of an attacker-controlled master; when the replica synchronizes, the server may load a new function context from the incoming RDB file, enabling arbitrary code execution.

Additional active CVEs on Redis 7.1.x:
| CVE | Severity | Vector |
|---|---|---|
| CVE-2026-23631 (DarkReplica) | **Critical** | Post-auth RCE via replica sync |
| CVE-2026-23479 | High | Use-after-free in unblock client flow |
| CVE-2026-25243 | High | Invalid memory access in RESTORE command |
| CVE-2026-25588 | High | RESTORE + RedisTimeSeries module |
| CVE-2026-23631 | Medium | Lua use-after-free via replica sync |

**All patched in Redis 7.4.9+ and 7.2.14+.**

**Mitigating factor:** Redis is Docker-internal, not publicly exposed, requires authentication. Application code does not use the RESTORE command. However, a compromised API or worker container could exploit DarkReplica internally.

**Recommended fix:** Change `redis:7-alpine` → `redis:7.4-alpine` (or `redis:7.2-alpine`) in `docker-compose.yml`. No API changes, no RQ changes needed. 1-line PR, assignable to Jules today.

### NL-to-Strategy UX Patterns (competitor analysis)

The 2026 no-code crypto trading market has converged on a common NL-guided pattern that validates the Blockbuilders NL wedge direction:

| Platform | NL Pattern | Relevance to Blockbuilders |
|---|---|---|
| **Vibe-Trading** (open source) | NLP prompt → market data loader → strategy → backtest → report | Direct competitor model; open source = reference implementation available |
| **Bitsgap AI Assistant** | Risk tolerance questionnaire → AI suggests bot configuration → user reviews before launch | "Draft-and-review" interaction model mirrors ADR-0012 disposition flow |
| **Gunbot AI** | Plain-text strategy prompts → custom strategy generation | Same NLP-to-strategy goal as #618 NL wedge |
| **PionexGPT** | Plain-English prompts → strategy configuration | Confirms user appetite for NLP at strategy creation, not just at backtest |

**Key pattern:** The most successful NL strategy UX in 2026 does NOT auto-apply the AI's output — it generates a reviewable draft that the user inspects and approves before running. This is exactly the `disposition flow` (ADR-0012) Blockbuilders has already built.

**Differentiation opportunity:** None of the competitors offer a **visual node graph** as the NL output. Blockbuilders' NL drafter outputs a structured graph that users can inspect and edit node-by-node — a richer post-draft experience than any competitor.
