## Executive Summary

The dependency landscape has materially shifted since the current stack baseline was set. The most urgent finding is a cluster of **critical-severity CVEs** in Redis, Starlette/FastAPI, PostgreSQL, and Next.js that require patching before the next production deploy. On the version-upgrade side, Next.js has moved to 16.2 (breaking async API changes), TypeScript 7.0 (Go-based, 10× compile speed) is mid-2026, and Tailwind v4.3 delivers 100× faster incremental builds. XyFlow v12 adds native dark mode and auto-pan, both relevant to the canvas. For open spec-ready issues, solid reference implementations now exist for PostHog A/B test bootstrapping (zero-flicker), Dagre/ELK auto-layout, and side-by-side equity curve comparison views.

| XyFlow | (unversioned) | 12.11.0 | Dark mode, auto-pan new |

**Next.js 16.2 key changes:**
- `params` and `searchParams` in page components are now **async** — must `await` them. Affects `strategies/[id]` and `strategies/[id]/backtest`.
- `cookies()`, `headers()`, `draftMode()` also async.
- Node.js 20.9+ required (18 dropped).
- Turbopack is now the default bundler (2–5× faster production builds, 5–10× faster Fast Refresh).
- New `"use cache"` directive and Partial Pre-Rendering (PPR) available.
- Node.js 20.9+ requirement means the Docker base image may need updating.

**React 19 highlights (already current):**
- `useOptimistic()` is well-suited for instant visual feedback when dragging nodes/edges on the canvas.
- `useActionState()` simplifies save-strategy and run-backtest server interactions.
- React Compiler (auto-memoisation) available in Next.js 16 — reduces re-renders on large canvas graphs.
- `ref` as direct prop (no `forwardRef` needed) — relevant when refactoring canvas node components.

**TypeScript 7.0 (mid-2026 — watch track):**
- Go-based compiler rewrite: 10× compile time, 8× faster project loading.
- Breaking: `strict` mode by default, ES5 target dropped, AMD/UMD/SystemJS removed.
- No action needed now; plan migration window after stable release.

**Tailwind CSS v4.3:**
- Configuration moves from `tailwind.config.js` to `@theme{}` CSS blocks — significant migration effort.
- 100× faster incremental builds with no new CSS; ~3.5× faster full builds.
- Native container queries (no plugin), 3D transform utilities, OKLCH color space.
- Practical for the canvas: container queries remove the need for `isMobileMode` prop drilling in some components.

**XyFlow v12.11.0:**
- `colorMode` prop (`"light"` | `"dark"` | `"system"`) — aligns with the existing dark mode toggle in DisplayContext.
- Auto-pan when dragging nodes near viewport edges — improves large-strategy editing UX.
- Better TypeScript generics for node/edge data shapes.
- Relevant to `CanvasMinimap.tsx` and `StrategyCanvas.tsx` dark mode integration.

- Dropped Python 3.9 support; requires Python 3.10+. Current Docker image uses Python 3.12 — no action needed.
- `sqlmodel-slim` distribution removed; standard `sqlmodel` only.
- New `.sqlmodel_update()` method useful for partial updates to `BacktestRun` and `User` models without full ORM cycles.

**PostgreSQL 18:**
- Native async I/O (io_uring) — up to 3× sequential scan throughput; significant for large candle data reads during backtesting.
- Skip scan for multi-column indexes — improves queries on `(asset, timeframe, timestamp)` index.
- Parallel GIN index builds — useful if `definition_json` JSONB columns are ever indexed.
- Docker image upgrade from `postgres:15` to `postgres:18` is straightforward; test migrations on 18 first.

**Redis 8.2.2:**
- Stream command extensions (`XDELEX`, `XACKDEL`) — useful for RQ job event tracking.
- Contains patches for two critical CVEs (see Security section). **Upgrade is security-mandatory.**

### 1.3 Spec-Ready Issues — Implementation References

The following issues are marked `📝 Spec Ready` in product.md. Research found concrete implementation guidance for each.

#### Multi-Period Batch Backtesting
- **Pattern:** Enqueue one RQ job per selected period; use a shared `batch_id` UUID to group results. Stream job completion events to the frontend via polling (current pattern) or SSE.
- **UX reference:** QuantConnect's backtest dashboard groups results by period with progressive fill-in; Portfolio Visualizer shows parallel equity curves for up to 3 runs.
- **Frontend:** Add preset checkbox strip (30d/60d/90d/120d/1y) above date picker; "Run All" button enqueues n jobs and opens a grouped results view. Each job counts toward daily limit.
- **Risk:** Daily limit enforcement must be atomic at enqueue time, not on completion.

#### Strategy Performance by Period Grid
- **Pattern:** Pre-compute period return from latest completed backtest run for each strategy × period; cache as a `strategy_period_cache` table keyed `(strategy_id, period_days)`.
- **UX reference:** Linear's issue list with sortable numeric columns is the interaction model. Color-code cells with green/red at ±0%.
- **Risk:** Stale cache if a strategy is edited after the cache is populated; invalidate on `POST /strategies/{id}/versions`.

#### Health Bar — Strategy Completeness Display
- **Reference:** The n8n visual workflow builder shows a persistent top bar with "nodes missing connections" counts using the same complete/warning/error tristate pattern.
- **Implementation note:** Client-side rule evaluation should mirror the existing validation endpoint rules in `backend/app/backtest/interpreter.py` — keep a single source of truth in a shared `validateStrategy()` TS utility called by both the Health Bar and the full Validate button.
- **Feature flag:** `canvas_flag_health_bar` in PostHog (consistent with existing SmartCanvas flag naming).

#### Inline Parameter Popover on Block Tap
- **Library:** Floating UI (`@floating-ui/react`) — handles collision-aware positioning, flip/shift middleware, and viewport clamping automatically. Already used indirectly via shadcn/ui Popover.
- **Reference:** Figma's property panel is the gold standard. VSCode's inline parameter hints show the compact-label live-update pattern.
- **Mobile fallback:** Below 768px, render a shadcn/ui `Sheet` half-screen instead of a popover. Share the same Inspector controls component between both surfaces.

#### Mobile Bottom Sheet Parameter Editing
- **Reference:** Maps.app (Google/Apple) bottom sheet pattern. Use `visualViewport` API to push content above the software keyboard.
- **Library:** shadcn/ui `Sheet` component with `side="bottom"`. No new dependency needed.

#### Template Educational Fields & Difficulty Ordering
- **Pattern:** Migration 029 already adds `teaches_description`, `difficulty`, and `sort_order`. Frontend change is additive: show badge chip on template card, hide section when `teaches_description` is null.
- **Risk:** Low; data-driven display change with no logic.

#### Storybook Framework
- **Reference:** Storybook 8.x with `@storybook/nextjs` preset. Requires zero-config setup with Next.js App Router via `storybookjs/storybook` v8.5+.
- **Constraint per product.md:** Must stay fully aligned with `docs/design-system.json` and `docs/design_concept.json`. Recommended approach: generate Storybook stories from design token files via a codegen step rather than maintaining them manually.

## 2. Security Risks

### CRITICAL — Patch immediately before next deploy

| CVE | Component | CVSS | Impact | Fix |
|---|---|---|---|---|
| CVE-2025-49844 "RediShell" | Redis 6.x/7.x/8.x | **10.0** | Authenticated RCE via Lua use-after-free; 60k+ instances exposed unauthenticated | Redis 8.2.2, 7.4.6, 7.2.11 |
| CVE-2026-23631 "DarkReplica" | Redis 7.2–8.6 | Critical | Authenticated RCE via replication subsystem use-after-free | Redis 7.4.9, 8.2.6, 8.6.3 |
| CVE-2026-48710 | Starlette / FastAPI | Critical | Host header bypass of host-validation; affects all FastAPI apps | Starlette ≥ 1.0.1; FastAPI 0.136.1 |
| CVE-2025-8715 | PostgreSQL 13–17 | Critical | `pg_dump` code injection → RCE as client OS user | PostgreSQL 18.3, 17.9 |

**Current Docker images `redis:7-alpine` and `postgres:15` are exposed to all four of these.**

### HIGH — Fix before next feature branch

| CVE | Component | Impact | Fix |
|---|---|---|---|
| CVE-2026-48526 | PyJWT | Algorithm confusion → forged tokens (auth bypass) | PyJWT ≥ 2.13.0 |
| CVE-2026-32597 | PyJWT | `crit` header validation bypass per RFC 7515 | PyJWT ≥ 2.13.0 |
| CVE-2026-23864 | Next.js | DoS via memory exhaustion on crafted requests | Next.js 16.2.5 |
| CVE-2026-23869 | Next.js App Router | Excessive CPU on crafted HTTP requests | Apply latest security patch |
| CVE-2025-8713 | PostgreSQL | Row-level security bypass via optimizer statistics | PostgreSQL 17.9+ |

**Project uses JWT HS256 signing (`backend/app/core/security.py`). PyJWT upgrade to ≥ 2.13.0 is mandatory.**

### MEDIUM — Address in next sprint

| CVE | Component | Impact | Fix |
|---|---|---|---|

### bcrypt note
CVE-2025-22228 (bcrypt truncation at 72 chars) affects Spring Security, not Python's `bcrypt` library. No action needed, but add a user-facing password length cap of 72 characters to prevent silently equivalent passwords.
