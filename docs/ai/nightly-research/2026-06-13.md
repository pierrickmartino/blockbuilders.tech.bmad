## 1. Findings for Active Issues

### 1.1 #649 (parent) / #651–#654 — Golden-backtest regression suite (ADR-0018, all `ready-for-agent`)

ADR-0018 (accepted 2026-06-12) extends the existing **engine-level** golden tests (`TestGoldenSnapshot`, `TestCharacterizationAllPaths` in `test_backtest_engine.py`) to the **full pipeline** (`validate_strategy` → `interpret_strategy` → `run_backtest`, mirroring `app/worker/jobs.py::run_backtest_job`), using the three seed templates already shipped in `app/data/strategy_templates.py`.

- **#651 (foundational)** — builds `run_golden_pipeline(definition_json, candles, run_config) -> BacktestResult`, a deterministic RSI-Oversold-Bounce candle series, and shared `mutate_tail` / `assert_prefix_unchanged` helpers for the look-ahead-bias check. Everything downstream depends on this one landing first and exporting stable helper names.
- **#652 / #653** — MA Crossover and Bollinger Breakout goldens, explicitly told to **reuse, not reimplement**, #651's helpers.
- **#654 — CI bootstrap.** Acceptance criteria: GitHub Actions workflow on `push`/`pull_request`, Python 3.12, install `backend/requirements.txt`, run `pytest` from `backend/`, no extra service containers (relying on `pytest.ini`'s SQLite/fake-Redis/S3/Stripe overrides).
  - **`.github/workflows/ci.yml` already does this** (and more — it also runs a frontend job with `npm ci && npm test && npm run lint && npm run build`). Its `backend` job: checks out, sets up Python 3.12 with pip cache, installs `requirements.txt`, runs `pytest tests/ -v` from `backend/`. This appears to satisfy #654's stated AC almost exactly.
  - The git history shows this workflow was added by a prior "rollout overnight factory" / "fix ci issue" commit pair, predating #654's filing — so #654 may be effectively done, or its AC may have been written without awareness of the existing file. Recommend a short triage pass (confirm the workflow currently passes on `main`, then either close #654 with a comment pointing at `ci.yml`, or narrow its scope to "verify golden tests added in #651–#653 run green under the existing workflow").
