## Executive Summary

Six days since the last report (2026-06-14). The main library and security developments:

**FastAPI** moved from 0.137.0 to **0.138.0** (released today, 2026-06-20), with 0.137.1 and 0.137.2 landing in between. The project is pinned to 0.136.1 — now **three minor versions behind**. The 0.137.x line introduced the `router.routes` tree-structure breaking change (confirmed safe for this codebase, which has no direct `.routes` iterations) and added the `iter_route_contexts()` migration escape hatch in 0.137.2. The 0.138.0 release adds `app.frontend()` for serving SPA static files, not relevant here. Upgrading to 0.137.2+ is low-risk and provides the new introspection utility.

**@xyflow/react** released **12.11.0** (~2026-06-01) with `autoPanOnSelection`, stricter `handleId`/`handleType` type enforcement, and several bug fixes including a node-resizing extent overflow. The project's `^12.10.0` semver range covers this release — it may already be installed depending on when `npm install` was last run. The `handleId` type-strictness change (error if `handleId` is used without `handleType`) could surface a TS error in existing canvas code if custom handles omit `handleType`.

**React 19.2.7** (FormData/Server Actions regression fix, released 2026-06-01) remains un-adopted — **5th consecutive report** noting this gap. No new React releases since.

**PostgreSQL 16.14** (released 2026-05-14) patched 11 CVEs including two CVSS 8.8 High vulnerabilities (CVE-2026-6473, CVE-2026-6475). The project's `postgres:16-alpine` floating tag should be pulling 16.14 if the image has been refreshed since mid-May; this is unconfirmed.

**Next.js** released 16.2.8 and 16.2.9 (both 2026-06-09) as packaging-only fixes with no functional changes. No security activity since the May 2026 coordinated patch.

On the active issues front: **#620** (configure `onboarding_ab` PostHog experiment) remains `ready-for-human` with no update — now 8 days since the routing fork went live (PR #622, 2026-06-12). **#618** (NL-wedge A/B harness) is `ready-for-agent` with a complete PRD. ## 1. Findings for Active Issues

### 1.1 #618 / #620 — NL-wedge onboarding A/B (ADR-0014)

**#620 is now 8 days stale** as `ready-for-human`. The `resolveOnboardingArm` routing fork landed in PR #622 (2026-06-12) and routes users to the NL-wedge or wizard based on the `onboarding_ab` PostHog flag — but the PostHog experiment itself has not been configured. Until the experiment is configured, `getExperimentVariant('onboarding_ab')` will return `undefined` for all users, which per the onboarding hook's safety net means all users land on the wizard (control) path. The 8-day window without a live experiment is a growing risk for the A/B validity window if beta users churn out of the new-user cohort.

### UX / Visual Strategy Builder research (2026)

Three converging trends in node-based visual editors relevant to the canvas roadmap:

1. **`autoPanOnSelection`** (just shipped in @xyflow/react 12.11.0) — auto-panning when dragging selections near viewport edge is now considered table-stakes UX. Relevant to the pending **#4.16 (Auto-layout / connection tidying)** and **#4.10 (Copy/paste blocks & subgraphs)** features where users must drag large selections.

2. **AI co-pilot within the editor** — industry-wide trend (Adobe Project Graph, Runway Workflows, 2026). The NL-wedge onboarding (#618) positions Blockbuilders ahead of most no-code fintech tools on this. No additional implementation reference beyond what ADR-0014 specifies.

3. **Responsible gamification in trading UX** — A 2026 behavioral-design study (Lollypop Design) documents that frequent-interaction patterns (auto-refresh, streak counters, real-time P&L) risk encouraging impulsive over-trading. Relevant to the **Strategy Monitor** (§7) and **Performance Alerts** (§9.10) features: alert frequency and auto-refresh cadence should be conservative defaults. The current design (daily scheduler, alert fires on auto-backtest only) already aligns with this principle.
