Separately, `npm audit` surfaced a **new moderate CVE (CVE-2026-41305 / GHSA-qx2v-qp2m-jg93)**: PostCSS <8.5.10 has an XSS-via-unescaped-`</style>` bug, and Next.js 16.2.x bundles a vulnerable PostCSS internally (confirmed via an upstream Next.js issue, auto-closed without a fix). On the **redis-py** side, the 7.x line picked up three June patch releases (7.2.2/7.3.1/7.4.1, all 2026-06-05) that bump a `pyjwt` floor to fix CVE-2026-32597 — **not applicable** here since the project doesn't install redis-py's optional `jwt` extra (auth uses `joserfc`) — and redis-py **8.0.0 went GA** (2026-05-28) with a RESP3-by-default switch and type-hint breaking changes, which is a planned-migration item, not urgent. No change on the Redis server CVE bundle (still fully patched by `redis:7.4.9-alpine`), `anthropic` SDK (still 0.109.1, no movement since 06-09), or PostgreSQL (still 18.4/17.10/16.14).
