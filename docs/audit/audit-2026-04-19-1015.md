# Repository Audit — Blockbuilders

- **Timestamp:** 2026-04-19 10:15 local
- **Repo:** blockbuilders.tech.bmad
- **Branch:** `design-system/signal-theme` (103 files changed vs `main`, +21,072 / −5,065)

## Overall assessment

The backend has a generally sound security/billing posture (bcrypt, Stripe webhook idempotency on `event_id`, constant-time reset token compare, OAuth state in Redis, explicit plan-tier gates), and the branch's frontend redesign is well-structured. The most serious issues are in **production deployment** (direct port exposure bypassing nginx, missing security headers, unauthenticated Redis) and one **billing correctness race** in backtest credit deduction. On the branch itself, issues are mostly localized: a couple of unguarded numeric expressions in the new backtest components, and a couple of design-system rule violations. Test coverage of the billing *deduction* path and cross-tenant access is the most notable gap.

No serious defects were discovered inside the core backtest engine or interpreter in this pass (limited depth — see *Not verified*).

### High

#### H1. Prod `api` container directly exposes port 8000, bypassing nginx

- **Location:** `docker-compose.prod.yml:6–7` (`api.ports: - "8000:8000"`)
- **Why it matters:** nginx provides the only rate limit (`limit_req_zone api_limit 10r/s`, `nginx.conf:7`), TLS, and (currently missing but expected) security headers. Publishing `8000:8000` on the host makes the FastAPI app reachable on plain HTTP without any of those protections — an attacker can hit `http://host:8000/*` directly and skip rate limiting and any forthcoming WAF/header policy.
- **Evidence:**
  ```yaml
  api:
    build: { context: ./backend, dockerfile: Dockerfile }
    ports:
      - "8000:8000"
  ```
- **Fix:** Remove the `ports:` block from the `api` service in prod and rely on the internal Docker network (`upstream api { server api:8000; }` already in `nginx.conf:10`). Do the same for `storage` (MinIO) if externally exposed.
- **Confidence:** High

#### H2. Race condition on backtest credit deduction

- **Location:** `backend/app/api/backtests.py:341–344` (single run) and `:547–550` (batch run)
- **Why it matters:** Credit check and decrement are non-atomic read-modify-write on `user.backtest_credit_balance`. Two concurrent requests from the same user can both see balance `≥ 1` and both deduct, letting the balance go negative (or a user get two runs for one credit). Non-financial surface is small (max a handful per user) but it's a direct billing correctness bug.
- **Evidence:**
  ```python
  # backtests.py:341
  if use_credit:
      user.backtest_credit_balance -= 1
      session.add(user)
      session.commit()
  ```
  Meanwhile `billing.py` already uses the correct atomic pattern (`update(User).where(...).values(balance=User.balance + N)`), so there's a clear in-repo precedent.
- **Fix:** Replace both sites with a single atomic guarded update, and reject when no row was affected:
  ```python
  result = session.execute(
      update(User)
      .where(User.id == user.id, User.backtest_credit_balance > 0)
      .values(backtest_credit_balance=User.backtest_credit_balance - 1)
  )
  session.commit()
  if result.rowcount == 0:
      raise HTTPException(status_code=402, detail="No backtest credits remaining.")
  ```
- **Confidence:** High

#### H3. nginx missing security headers (HSTS, frame/content-type protections)

- **Location:** `nginx.conf:20–48` (HTTPS server block)
- **Why it matters:** The app stores JWTs in `localStorage` (`frontend/src/context/auth.tsx`), so XSS exposure is already the dominant token-theft risk. Without HSTS + `X-Content-Type-Options: nosniff` + `X-Frame-Options` / `frame-ancestors`, the app is susceptible to SSL-strip on first visit, MIME-sniff XSS vectors, and clickjacking.
- **Evidence:** `nginx.conf` sets proxy, TLS, rate limit — no `add_header` directives at all.
- **Fix:** Add to the `listen 443 ssl` server block:
  ```nginx
  add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
  add_header X-Content-Type-Options "nosniff" always;
  add_header X-Frame-Options "DENY" always;
  add_header Referrer-Policy "strict-origin-when-cross-origin" always;
  ```
  Consider a CSP once inline-script usage is enumerated.
- **Confidence:** High

#### H4. Prod Redis runs without authentication

- **Location:** `docker-compose.prod.yml:96–105`
- **Why it matters:** Redis holds OAuth state (`oauth_state:{state}`, `auth.py:203`), password-reset rate-limit counters (`auth.py:44–67`), and the RQ job queue. The service has no `--requirepass` and no bind restriction. Any container on the compose network (or anyone who accidentally exposes `6379`) can read/modify OAuth state and enqueue jobs. Defense-in-depth problem even behind a VPC.
- **Evidence:**
  ```yaml
  redis:
    image: redis:7-alpine
    volumes: [redis_data:/data]
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
  ```
- **Fix:** Add a password and pass it through REDIS_URL:
  ```yaml
  redis:
    command: ["redis-server", "--requirepass", "${REDIS_PASSWORD}"]
    healthcheck:
      test: ["CMD", "redis-cli", "-a", "${REDIS_PASSWORD}", "ping"]
  # api/worker/scheduler: REDIS_URL=redis://:${REDIS_PASSWORD}@redis:6379/0
  ```
- **Confidence:** High

#### H5. `KPIStrip` "trades per month" subtitle can render `NaN` when timestamps are missing

- **Location:** `frontend/src/components/backtest/KPIStrip.tsx:79`
- **Why it matters:** If any of `trades[0].entry_time` / `trades[last].exit_time` is `undefined`/empty, `new Date(undefined).getTime()` returns `NaN`, and the subtitle renders `"NaN per month"`. Silent data corruption in the UI violates CLAUDE.md's "warn users, never silent fail" rule for derived data.
- **Evidence:**
  ```tsx
  subtitle={trades.length > 0
    ? `${Math.round(summary.num_trades / Math.max(1, Math.ceil(
        (new Date(trades[trades.length - 1]?.exit_time).getTime()
         - new Date(trades[0]?.entry_time).getTime()) / (30 * 86400000))))} per month`
    : "—"}
  ```
  Optional chaining only guards property access, not the `getTime()` call on an Invalid Date.
- **Fix:** Extract and guard:
  ```tsx
  const first = trades[0]?.entry_time;
  const last = trades[trades.length - 1]?.exit_time;
  const spanMs = first && last ? new Date(last).getTime() - new Date(first).getTime() : NaN;
  const months = Number.isFinite(spanMs) && spanMs > 0 ? Math.max(1, Math.ceil(spanMs / (30 * 86400000))) : null;
  const subtitle = months ? `${Math.round(summary.num_trades / months)} per month` : "—";
  ```
- **Confidence:** High

### Medium

#### M1. `reset_token` index is not unique

- **Location:** `backend/alembic/versions/007_add_auth_fields.py:33`; model `backend/app/models/user.py:92`
- **Why it matters:** Reset tokens are generated via `secrets.token_urlsafe(32)` so collision is practically nil, but the schema gives no DB-level guarantee of uniqueness. If an application bug ever reused a token, `confirm_password_reset` (`auth.py:177`) would `.first()` an arbitrary matching row and reset that user's password. This is pure defense-in-depth, but it's cheap.
- **Evidence:** `op.create_index("ix_users_reset_token", "users", ["reset_token"], unique=False)`.
- **Fix:** New migration:
  ```python
  op.drop_index("ix_users_reset_token", table_name="users")
  op.create_index("ix_users_reset_token", "users", ["reset_token"], unique=True)
  ```
  (Postgres allows multiple NULLs in a unique index, so existing data is safe.)
- **Confidence:** High

#### M2. No healthchecks on `api` / `worker` / `scheduler` in prod

- **Location:** `docker-compose.prod.yml:2–78`
- **Why it matters:** `restart: always` will restart a crashed process, but a stuck or deadlocked uvicorn/worker will keep running and fail silently. `api` has a `/health` endpoint used by nginx already (`nginx.conf:44`); add it to compose so `restart` and (future) orchestration can see it.
- **Fix:**
  ```yaml
  api:
    healthcheck:
      test: ["CMD", "curl", "-fs", "http://localhost:8000/health"]
      interval: 15s
      timeout: 3s
      retries: 3
  ```
  Add an equivalent check (even a simple `python -c "..."` pinging Redis) to worker/scheduler.
- **Confidence:** High

#### M3. Design-system rule violation: hardcoded slate colors in `TransactionCostAnalysis`

- **Location:** `frontend/src/components/TransactionCostAnalysis.tsx` (segment + legend for "Spread")
- **Why it matters:** `frontend/CLAUDE.md` explicitly requires CSS variables from `globals.css` (`--primary`, `--muted`, …) and forbids arbitrary palette values. Hardcoding `bg-slate-400 dark:bg-slate-500` (a) breaks the theme when `design-system.json` changes, (b) is inconsistent with adjacent segments that already use `bg-destructive` / `bg-warning`.
- **Evidence:**
  ```tsx
  <div className="bg-slate-400 transition-all dark:bg-slate-500" style={{ width: `${spreadPct}%` }} />
  // …
  <LegendDot colorClass="bg-slate-400 dark:bg-slate-500" label={`Spread ${formatNumber(spreadPct, 1)}%`} />
  ```
- **Fix:** Map "spread" to a semantic token already in the design system (e.g. `bg-muted-foreground/60`) or introduce one. Keep the fix local — don't add new global utilities.
- **Confidence:** High

#### M4. `api` container publishes to host in prod even though nginx sits in front

- Consolidated into H1; keeping as a pointer so the MinIO `storage` service is double-checked before deploy. **Not verified** whether MinIO is published externally; if it is (e.g. via a separate compose override), apply the same fix.
- **Confidence:** Medium

### Low

#### L1. Pagination buttons lack accessible labels

- **Location:** `frontend/src/components/backtest/TradesSection.tsx` (pagination control, ~`:320–340`)
- **Why it matters:** Numeric-only button children (`<button>{page}</button>`) read as "1", "2", "3" to screen readers with no indication that they control pagination or which page is current. WCAG 2.1 A.
- **Fix:** Add `aria-label={\`Go to page ${page}\`}` and `aria-current={page === currentPage ? "page" : undefined}`.
- **Confidence:** High

#### L2. `DistributionRow` uses array index as React key for `<Cell>`

- **Location:** `frontend/src/components/backtest/DistributionRow.tsx:165–180` (and `:227–237`)
- **Why it matters:** Index keys are fine while bucket ordering is static (the current case), but this is a known fragility trap — a future sort change would silently misalign bar colors.
- **Fix:** Use `key={entry.label}` (labels are already unique within each distribution).
- **Confidence:** High

#### L3. `PageHeader.formatHeaderDate` accepts out-of-range month/day

- **Location:** `frontend/src/components/backtest/PageHeader.tsx:23–35`
- **Why it matters:** A string like `"2025-13-40"` passes the `!year || !month || !day` guard and produces a silently wrong rendered date via `Date` normalization. Bad data in, bad data out — no crash but misleading UI.
- **Fix:** Validate against `/^\d{4}-\d{2}-\d{2}$/` and enforce `1 ≤ month ≤ 12`, `1 ≤ day ≤ 31`; fall back to the raw string on failure.
- **Confidence:** Medium

## Merge recommendation

**Request changes.**

The branch itself (design refresh + backtest redesign) is in good shape, but three items block a clean merge:

1. **H2** — atomic credit deduction is a real billing correctness bug already live on `main`; since this branch is being shipped, include the fix here.
2. **H5** — `NaN per month` in the new KPI strip is branch-local and visible to every user.
3. **M3** — hardcoded slate colors violate a rule the branch itself is meant to enforce (it *is* the design-system branch).

**H1, H3, H4, M2** are production infra changes that should ship in a separate hardening PR but are called out here because they are the highest-impact items in the repo today.
