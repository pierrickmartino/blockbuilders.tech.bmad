## Overall Impression

The scaffolding is disciplined — mono numbers, flat cards, semantic palette. The **completed-result view** is the weak spot: 11 bordered sections of near-identical weight (KPI strip → equity+runs → drawdown → position + seasonality → two distribution cards → trades → TCA with another 14 mini-tiles), each screaming at `text-[15px] font-semibold`. The page ships a lot; it doesn't *say* anything. Biggest opportunity: **kill visual equality across sections**, introduce tiers (hero / analysis / reference), and let the equity curve + 3 primary KPIs own the first screen.

## What's Working

1. **StatusBadge is the right abstraction** (`StatusBadge.tsx:36–54`) — semantic color, animated ping on `running` only, ghost neutral for `skipped`, reused across header and runs list. One source of truth.
2. **DataAvailabilitySection auto-expands on issues** (`DataAvailabilitySection.tsx:124`) — warnings aren't behind a click. Exactly the trust move Signal demands.
3. **Numbers really are first-class.** `globals.css:113–127` forces mono on tabular figures and recharts ticks; `PerfCell` + KPI subtitles respect it. Strongest Signal win on the page.

## Priority Issues

### [P0] Completed-result view has no hierarchy
- **What**: 11 equal-weight cards stacked (`page.tsx:1234–1517`). Same radius, same h2 size, same padding. No tiers.
- **Why**: Alex scrolls ~3000px to compare runs with no landmarks. Jordan drowns. Reads as "11 widgets a backend team shipped."
- **Fix**: Three tiers. Tier 1 (hero): KPI strip + equity curve — no card chrome, sits on page bg, `border-t/b` only, 3 primary KPIs at ~26px mono dominate. Tier 2 (analysis): drawdown / position / seasonality / distributions — current card treatment. Tier 3 (reference): trades + TCA — `bg-secondary/40`, smaller h2, collapsed by default.
- **Commands**: `/layout` then `/distill`.

### [P0] "Where do I decide next?" is split across the page
- **What**: `RunConfig` below `DataAvailability`, above results; `BacktestRunsList` as sidebar inside results (`page.tsx:1350–1367`). The two primary actions — pick next backtest, pick run to examine — are 6 sections apart.
- **Why**: Every return visit forces Alex past config to find recent runs. Runs-next-to-equity is good when comparing; wrong on arrival.
- **Fix**: Option A — demote `RunConfig` to a slide-in sheet triggered by the header "Run backtest" button (already exists, `PageHeader.tsx:213–222`); promote recent runs to a summary band under header. Option B — sticky right-rail for runs that stays through the scroll.
- **Commands**: `/shape` (IA rework), `/layout`.

### [P1] Card chrome drift
- **What**: `rounded` / `rounded-md` / `rounded-lg` / `rounded-xl` / `rounded-full` coexist on structural containers. Padding permutations: `px-3 py-2`, `px-4 py-3`, `px-4 py-4`, `px-4 py-4 sm:px-5`, `px-5 py-4`, `px-4 py-5`, `px-6 py-6`.
- **Why**: This is the single loudest "AI-cobbled components" tell. Signal = discipline = one radius, three padding tiers.
- **Fix**: Pick one radius (suggest `rounded` 4px). Three padding tiers — `p-3` dense, `p-4 sm:p-5` normal, `p-6` hero. Remove `rounded-xl` / `rounded-full` from containers (keep on dots/avatars).
- **Commands**: `/polish`.

### [P1] Jargon with no escape hatch
- **What**: Sortino / Calmar in KPI subtitle (`KPIStrip.tsx:60`), SKEW badge (`DistributionRow.tsx:133`), return-distribution Y axis unlabeled. No tooltips, no inline help.
- **Why**: "Match system ↔ real world" fail. Retail trader ≠ institutional quant — plain-language needed.
- **Fix**: Wire the existing `InfoIcon` pattern (already in `DataAvailabilitySection`) onto every KPI card and the SKEW badge. 1-sentence plain definition + 1-sentence formula/assumption. Add axis label "Trades" on both distribution charts.
- **Commands**: `/clarify`.

### [P1] Four alert styles
- **What**: Error banner `bg-destructive/5` (`page.tsx:1101`), status banner `bg-green-50/950` (`page.tsx:1110`), availability warning `amber-300` (`RunConfig.tsx:175`), `LowTradeCountWarning` `warning/30 bg-warning/10`. Four conventions for the same job.
- **Why**: Inconsistent recovery signals; drifts off semantic tokens.
- **Fix**: One `<PageAlert variant="info|success|warning|destructive">` primitive on semantic tokens. Migrate all four call sites.
- **Commands**: `/polish`.

### [P2] No cancel / undo for runs
- **What**: Once batch POSTed (`page.tsx:894–931`), no UI to stop a pending/running run or cancel the remaining queue.
- **Why**: Wrong-period fires cost compute and minutes. Disciplined workstation must have a stop.
- **Fix**: `Cancel` action on `StatusBadge` hover for pending/running rows; "Pause batch" in status banner while `activeBatchId` set. Requires backend cancel endpoint.
- **Commands**: `/harden`.

## Persona Red Flags

**Alex — disciplined retail trader, multi-hour sessions**
- 4+ scroll actions to find recent runs on every return visit.
- No keyboard nav of runs list. Mouse-only selection for someone running 20 backtests/afternoon.
- Costs column in trades table (`TradesSection.tsx:280–285`) redundant with TCA section below.
- `selectedRunIds` resets on reload (`page.tsx:494`) → cross-session pinning lost.

**Jordan — first-time backtester**
- KPI strip bombs them with 6 unfamiliar terms, zero tooltips.
- `WhatYouLearnedCard` — the one warm moment — gates on `has_completed_onboarding` + browser localStorage (`page.tsx:124–136`). Dismissed once = never seen again, even on a different device.
- Distribution charts' Y axis unlabeled → Jordan can't decode "trade count."
- Header "Run backtest" silently disabled when no periods selected (`PageHeader.tsx:215`). No hover helper text.
- Seasonality heatmap is color-only red/green (`page.tsx:191–215`). Colorblind Jordan gets nothing.

## Provocative Questions

1. **Does the page need 11 sections, or 3?** What if primary = hero metric + equity curve behind it + runs list, and drawdown / position / seasonality / distributions / trades / TCA live under an "Analyze further" tab bar?
2. **Does `KPIStrip` need to be cards at all?** A single horizontal bar — labels above, mono numbers below, dividers between, no borders — reads more Bloomberg, less SaaS. Frees the equity chart to breathe.
3. **Does Jordan need Sortino + Calmar on screen 1?** First-run could show `Total return | Win rate | Max DD | Trades` only; quant row revealed on return visits.
4. **Why is `RunConfig` inline above results rather than a slide-in sheet?** The header already has the Run button. The inline form is redundant on return.
5. **The seasonality gloss is the one un-Signal flourish — is it load-bearing or is it the one thing that slipped past?** If flat looked boring, fix contrast, not gloss.
