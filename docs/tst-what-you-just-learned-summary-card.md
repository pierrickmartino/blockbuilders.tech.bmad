# TST: What You Just Learned Summary Card

- [ ] Seed/create a user with exactly one completed backtest and open that run’s results page.
- [ ] Verify a card titled **What you just learned** is visible below the results area.
- [ ] Verify card body is 1–2 sentences (not a paragraph block) and reads in plain language.
- [ ] Verify card text includes strategy-vs-buy-and-hold comparison wording.
- [ ] Verify card text includes a percentage-point delta value derived from strategy and benchmark returns.
- [ ] Verify card does not appear above charts/summary (must be below results content as specified).
- [ ] Seed/create a user with two or more completed backtests and open latest results page.
- [ ] Verify the “What you just learned” card is not rendered for second+ backtest cases.
- [ ] Refresh the first-run results page and confirm behavior remains correct (still shown only for eligible first-ever case).
- [ ] Validate responsive behavior on mobile and desktop breakpoints (card remains readable and correctly positioned).
- [ ] Validate fallback behavior when required metrics are missing (card suppressed or safe fallback copy, no page crash).
- [ ] Run frontend lint/type-check and confirm no new warnings/errors introduced by the feature.
